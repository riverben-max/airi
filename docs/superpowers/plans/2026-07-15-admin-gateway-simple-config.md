# 管理后台模型接入简化实施计划

## 背景

官方聊天接入表单将路由模型 ID、上游模型、Key 条目 ID、展示名称、Flux 和默认
模型开关同时暴露给管理员。对于 OpenAI 兼容服务，管理员通常只有基础 URL 和
API Key；此外，在系统尚未设置默认聊天模型时，不勾选“设为默认”会因别名同步
读取不到默认模型而导致整个保存流程失败。

## 实施范围

1. 默认表单只突出基础 URL、API Key 和自动发现的模型。
2. 将路由 ID、Key 条目、计费与状态选项收进高级设置。
3. OpenAI 兼容根 URL 自动补全标准 `/v1` 路径。
4. 自动发现模型后优先复用当前模型；必要时自动补齐路由 ID 和展示名称。
5. 未设置默认模型且未勾选默认时，跳过依赖默认模型的别名同步，但仍保存路由和计费模型。

## 验收标准

- 只填写根 URL 和 API Key 时能发现模型并形成可保存配置。
- 不勾选默认模型时保存成功，且不修改 `DEFAULT_CHAT_MODEL`。
- 勾选默认模型时继续写入默认值并同步 `auto` 别名。
- 管理后台 API 单元测试、Web 类型检查和生产构建通过。
- 生产页面展示简化表单，现有聊天配置和对话不受影响。
