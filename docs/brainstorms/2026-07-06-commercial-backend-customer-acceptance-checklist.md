# AIRI 商业化后端客户验收清单

本清单面向客户和验收人员，用于对照商业化后端、订阅、支付、官方 provider 和中文 UI 的交付结果。验收前请准备测试账号、管理员账号、可访问的测试/联调环境，以及不含密钥明文的部署配置说明。

## 验收环境说明

- Cloudflare Tunnel 仅用于临时联调和演示，不作为正式上线环境；Tunnel 重启后域名可能变化，需要同步更新 EPAY_CALLBACK_BASE_URL 并重新生成新的支付订单。
- 正式上线至少需要：2 核 4G 服务器、可公网访问的正式域名、服务端正式部署地址、前端正式部署地址、正式 `EPAY_CALLBACK_BASE_URL`。
- 真实 Epay 支付必须使用客户自己的商户号和商户密钥；验收材料不得记录真实密钥，也不得截图完整敏感配置。
- 支付发货以服务端异步 `notify_url` 回调验签结果为准；浏览器 `return_url` 只用于返回前端展示和刷新状态。

状态取值：待验收 / 已通过 / 阻塞。

需求文档中的 `[x]` 只表示实现和本地验证已完成，不等于客户验收已通过。本清单的状态必须依据截图、接口响应、DB 检查、Network 记录或真实支付回调等证据改为“已通过”；没有证据时不得仅凭需求文档完成状态更新。

## 仍需人工验收项

- 真实 0.01 元 Epay 支付：需要客户真实商户配置、可公网访问的服务端回调地址和验收付款操作。
- Epay `notify_url` 异步回调：需要确认服务端收到真实 Epay 回调、验签成功、订单状态更新、订阅生效、Flux 发放且不重复。
- 前端浏览器支付返回：需要在真实浏览器完成付款后验证 return 跳转、订阅/Flux 刷新和中文提示。
- 正式上线环境：需要把临时 Cloudflare Tunnel 替换为正式 2 核 4G 服务器、公网域名和正式 `EPAY_CALLBACK_BASE_URL`。
